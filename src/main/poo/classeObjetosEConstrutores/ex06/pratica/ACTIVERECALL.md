# Active Recall - toString

## Perguntas e Respostas

### Questão 1: Comportamento Padrão do toString

**Pergunta:** (Verdadeiro ou Falso) Se eu não criar o método toString, o Java imprime um código estranho que representa o endereço de memória do objeto (ex: Personagem@2f92e0f4).

**Resposta:** Verdadeiro

---

### Questão 2: Anotação @Override

**Pergunta:** (Múltipla Escolha) O que a anotação @Override faz?

a) Deixa o código mais rápido.

b) Indica que estamos substituindo um método padrão do Java por uma versão nossa.

c) Bloqueia o acesso ao método.

**Resposta:** b) Indica que estamos substituindo um método padrão do Java por uma versão nossa.

---

### Questão 3: Objetivo do toString

**Pergunta:** (Verdadeiro ou Falso) O objetivo do toString é mostrar os dados do objeto em formato de texto (String).

**Resposta:** Verdadeiro
